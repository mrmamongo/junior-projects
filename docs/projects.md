# 10 проектов для Junior Python-разработчика

> **Ограничения:** чистый Python 3.11+, стандартная библиотека, без фреймворков.
> Каждый проект решается за 2-7 дней и даёт видимый, реально используемый результат.

---

## 1. Интерактивная CLI-игра «Приключение в терминале»
**Сложность:** 1/5
**Тип:** Консольная утилита
**Чему учит:** `argparse`, работа с файлами (`pathlib`), парсинг JSON, словари, циклы, функции.
**Описание:** Текстовый квест, где сюжет, локации и варианты выбора хранятся в JSON-файле. Игрок вводит номер действия, игра подгружает следующую сцену. Сюжет можно редактировать, не трогая код.
**MVP-фичи:**
1. Загрузка сценария из JSON (локации, описания, переходы).
2. Интерактивный выбор действий по номеру.
3. Сохранение и загрузка прогресса в файл.
4. Простой инвентарь (предметы влияют на доступные действия).
5. Цветной вывод через `rich` *(единственная внешняя библиотека — опционально, можно `colorama` или ANSI-коды)*.
**Расширения:**
- Условные переходы на основе инвентаря и флагов.
- Система битв с простым RNG (`random`).
- Генератор сценариев из директории с Markdown-файлами.

## 2. Утилита резервного копирования «BackupBuddy»
**Сложность:** 2/5
**Тип:** Консольная утилита
**Чему учит:** `shutil`, `pathlib`, `argparse`, `hashlib`, `zipfile`, работа с деревом файлов.
**Описание:** Консольная утилита, которая создаёт архив `.zip` указанной директории, пропуская ненужные файлы по шаблону (как `.gitignore`). Проверяет контрольные суммы и не копирует неизменённые файлы.
**MVP-фичи:**
1. Рекурсивное копирование директории в ZIP-архив с сохранением структуры.
2. Фильтрация файлов по списку исключений (чёрный список расширений/папок).
3. Сравнение по `hashlib.md5` — архивирует только изменённые или новые файлы.
4. CLI-интерфейс с `argparse` (`--source`, `--dest`, `--exclude`).
5. Вывод прогресса в терминал (количество файлов, размер).
**Расширения:**
- Инкрементальные бэкапы с хранением версий (backup_001.zip, backup_002.zip).
- Восстановление из бэкапа с распаковкой в указанную директорию.
- Планировщик через `sched` или интеграция с cron.

## 3. Парсер и анализатор логов веб-сервера
**Сложность:** 3/5
**Тип:** Консольная утилита
**Чему учит:** `re` (регулярные выражения), `collections.Counter`, генераторы, `argparse`, работа с большими файлами.
**Описание:** Утилита анализирует логи в формате Nginx/Apache, извлекает IP, статус-коды, URL и временные метки. Выводит топ посещаемых страниц, ошибки 4xx/5xx, подозрительные IP с частыми запросами.
**MVP-фичи:**
1. Парсинг строк лога через регулярное выражение (IP, метод, URL, статус, размер, User-Agent).
2. Подсчёт запросов по статус-кодам (2xx, 3xx, 4xx, 5xx).
3. Топ-10 самых посещаемых URL и топ-10 IP по количеству запросов.
4. Обнаружение "подозрительной активности": IP с >N запросов за минуту.
5. Отчёт в текстовом виде + экспорт в JSON.
**Расширения:**
- Фильтрация по временному диапазону (`--from`, `--to`).
- Анализ User-Agent: распределение по браузерам/ботам.
- Визуализация временного ряда запросов (ASCII-график в терминале).

## 4. Простой файлообменник «FileDrop» (HTTP + Socket)
**Сложность:** 3/5
**Тип:** Веб-сервис
**Чему учит:** `http.server`, `socketserver`, HTML-формы, multipart-разбор вручную, `mimetypes`, потоковая передача данных.
**Описание:** Минималистичный веб-сервис на `http.server` для загрузки и скачивания файлов в локальной сети. Генерирует HTML-страницу со списком файлов и формой загрузки. Полезен для быстрого обмена между устройствами в одной сети.
**MVP-фичи:**
1. Сервер на `http.server.HTTPServer` с кастомным `BaseHTTPRequestHandler`.
2. GET `/` — HTML-страница со списком файлов и ссылками для скачивания.
3. POST `/upload` — приём файла через HTML-форму, сохранение в указанную директорию.
4. GET `/download?name=filename` — скачивание файла с корректным `Content-Type`.
5. Базовая защита: проверка имени файла (`pathlib.Path.sanitize`), ограничение размера.
**Расширения:**
- Прогресс-бар загрузки через chunked transfer или SSE (Server-Sent Events) на чистом HTTP.
- Аутентификация по простому токену в cookie/query string.
- Drag-and-drop интерфейс на чистом HTML5 + JS (фронтенд минимальный).

## 5. Консольный чат-сервер «CocketChat»
**Сложность:** 3/5
**Тип:** Веб-сервис
**Чему учит:** `socket`, `selectors` (или `asyncio`), многопоточность/многозадачность, протоколы поверх TCP, кодирование/декодирование.
**Описание:** Текстовый чат: сервер рассылает сообщения всем подключённым клиентам. Клиенты консольные. Реализуется простой текстовый протокол поверх TCP. Поддержка никнеймов и комнат.
**MVP-фичи:**
1. TCP-сервер, принимающие несколько клиентов через `selectors` (или `asyncio.start_server`).
2. Рассылка сообщений всем участникам (broadcast).
3. Простой текстовый протокол: `NICK alice`, `MSG Привет всем!`, `QUIT`.
4. Консольный клиент: ввод сообщений в одном потоке, чтение сервера в другом (`threading`).
5. Список участников по команде `WHO`.
**Расширения:**
- Приватные сообщения: `MSG @bob секрет`.
- Комнаты/каналы: `JOIN #python`.
- История сообщений с сохранением в JSON.

## 6. Многопоточный сканер портов «PortHawk»
**Сложность:** 3/5
**Тип:** Консольная утилита
**Чему учит:** `socket`, `concurrent.futures.ThreadPoolExecutor`, `argparse`, таймауты, базовая сетевая диагностика.
**Описание:** Быстрая проверка открытых TCP-портов на заданном хосте. Использует пул потоков для параллельного сканирования. Выводит список открытых портов с предполагаемыми сервисами (на основе распространённых портов).
**MVP-фичи:**
1. Сканирование диапазона портов с таймаутом 0.5-1 сек.
2. Параллельное сканирование через `ThreadPoolExecutor`.
3. Распознавание типовых сервисов (22 — SSH, 80 — HTTP, 443 — HTTPS и т.д.).
4. Вывод результатов в таблицу: порт, статус (open/closed/filtered), сервис.
5. Параметры CLI: `--host`, `--ports` (диапазон или список), `--threads`, `--timeout`.
**Расширения:**
- SYN-сканирование (полуоткрытое) через `socket.SOCK_RAW` *(требует прав root, для Linux)*.
- Определение ОС по TTL и баннеру (banner grabbing).
- Экспорт в JSON + сравнение двух сканов для выявления изменений.

## 7. Утилита синхронизации директорий «SyncIt»
**Сложность:** 4/5
**Тип:** Консольная утилита
**Чему учит:** `hashlib`, `pathlib`, рекурсивный обход деревьев, сравнение структур данных, `shutil`, `argparse`.
**Описание:** Двусторонняя синхронизация двух директорий по контрольным суммам. Находит добавленные, изменённые и удалённые файлы. Режимы: зеркалирование (A → B), двусторонний мерж, сухой прогон (только показать, что изменится).
**MVP-фичи:**
1. Рекурсивный расчёт `hashlib.sha256` для всех файлов в обеих директориях.
2. Построение "снимка" директории: словарь `{относительный_путь: хеш}`.
3. Определение различий: новые, изменённые, удалённые, переименованные (по хешу).
4. Три режима работы: `--mirror`, `--two-way`, `--dry-run`.
5. Копирование/удаление файлов с сохранением структуры и метаданных.
**Расширения:**
- Игнорирование файлов по `.syncignore` (аналог `.gitignore`).
- Сетевая синхронизация: клиент-сервер через `socket` с передачей хешей и файлов.
- Конфликт-резолюция: при двусторонней синхронизации файла с разными версиями — сохранить оба с меткой времени.

## 8. Консольный рендерер Markdown «TermMark»
**Сложность:** 4/5
**Тип:** Консольная утилита
**Чему учит:** парсинг и токенизация, конечные автоматы/состояния, регулярные выражения, форматирование ANSI-вывода.
**Описание:** Конвертирует Markdown-файлы в красиво отформатированный текст для терминала. Поддерживает заголовки, списки, жирный/курсивный текст, цитаты, кодовые блока и ссылки. Не использует внешние парсеры — парсер пишется с нуля.
**MVP-фичи:**
1. Парсер Markdown с нуля: разбор строки за строкой, определение блоков (heading, list, code, paragraph).
2. Рендеринг в ANSI-терминал: заголовки цветные и подчёркнутые, списки с маркерами, код на сером фоне.
3. Поддержка строчных элементов: `**жирный**`, `*курсив*`, `` `код` ``, `[ссылка](url)`.
4. Обёртка текста по ширине терминала (`os.get_terminal_size`).
5. Чтение из файла или `stdin`, вывод в `stdout`.
**Расширения:**
- Поддержка таблиц Markdown (выравнивание столбцов).
- Нумерованные списки и вложенные списки с отступами.
- "Слайд-режим": по страницам с навигацией (как `less`).

## 9. Микро-веб-фреймворк и блог «NanoSite»
**Сложность:** 5/5
**Тип:** Веб-сервис
**Чему учит:** `http.server`, роутинг, шаблонизация через `string.Template`, MIME-типы, cookie/sessions через `http.cookies`, статические файлы.
**Описание:** Пишем свой минимальный веб-сервер с роутингом и шаблонами — без Flask/Django. На его основе — простой блог: главная со списком постов, страница поста (из Markdown), форма добавления комментария. Всё хранится в JSON-файлах.
**MVP-фичи:**
1. Кастомный роутер: декораторы `@route("/", method="GET")` поверх `http.server`, сопоставление путей и вызов обработчиков.
2. Шаблонизатор на `string.Template` + собственная логика `{% for %}`, `{% if %}` (или упрощённый вариант подстановки переменных).
3. Блог: главная (`/`) — список постов; `/post/<id>` — страница поста; `/post/<id>/comment` — POST-обработка формы.
4. Хранение постов и комментариев в JSON-файлах (база данных — файлы).
5. Статические файлы: CSS через `http.server` с правильными `Content-Type`.
**Расширения:**
- Сессии через подписанные cookie (`hmac` + секретный ключ).
- Простая админка с HTTP Basic Auth (`Authorization` header).
- RSS/Atom-лента через генерацию XML (`xml.etree.ElementTree`).

## Рекомендуемый порядок прохождения

**Неделя 1-2:** Начните с проектов 1-3. Они дают уверенность, покрывают базовые темы и ощутимый результат.

**Неделя 3-4:** Переходите к проектам 4-7. Здесь появляется сеть, регулярные выражения и многопоточность — ключевые навыки для junior.

**Неделя 5-7:** Проекты 8-9 — для закрепления. NanoSite особенно полезен: вы поймёте, как работают "настоящие" фреймворки изнутри.

> **Совет:** Для каждого проекта создавайте `README.md` с инструкцией запуска и примерами. Это тренирует документирование кода — важный навык в командной работе.
